Diabetes Readmission
Preprocessing in Excel

Steps

  1. Dropping Unnecessary Columns: You identified and removed columns that were not relevant to your analysis or had a high percentage of missing values. These columns included:
    • Weight: Many values were missing, represented by ?, and it was deemed not critical for the analysis.
    • Payer Code: This column was likely deemed irrelevant for the analysis.
    • Medical Specialty: This column had a high percentage of missing values.
  2. Handling Missing Values: You addressed missing values to ensure data completeness and integrity:
    • Replaced ? with appropriate values or NaN to identify missing entries.
    • Dropped rows with missing values in critical columns to maintain data quality.
  3. Encoding Categorical Variables: You converted categorical variables into numerical representations to facilitate analysis. This included:
    • Readmitted: Converted categorical values (NO, >30, <30) into numerical values.
    • Change and DiabetesMed: Converted No and Yes into binary values.
    • Gender: Converted Female and Male into binary values.
    • Age: Converted age ranges into numerical categories.
    • Race: Mapped racial categories to numerical values.
    • Admission Type, Discharge Disposition, and Admission Source: Mapped these categorical values to numerical codes.
    • Medication Columns: Encoded values indicating whether a medication was used (No, Steady, Up, Down) into binary or numerical values.
  4. Standardizing Data: You ensured consistency in data formatting across the dataset:
    • Standardized categorical text by converting all entries to a consistent format (e.g., lowercase or uppercase).
    • Ensured that age ranges were consistently formatted and represented.
  5. Removing Duplicates: You removed duplicate rows to ensure data integrity:
    • Identified and removed duplicate rows based on unique identifiers such as encounter_id to prevent redundancy.
  6. Filtering Out Irrelevant Data: You filtered out records with unrealistic or irrelevant values to improve data quality:
    • Removed records with unrealistic values in critical columns, such as very high time_in_hospital values that were deemed implausible.
    • Ensured that the dataset included only relevant patient records for the analysis.

Summary

Through these detailed steps, you transformed the raw data into a clean, analyzable format by dropping unnecessary columns, handling missing values, encoding categorical variables, standardizing data, removing duplicates, and filtering out irrelevant data. This process ensured that the dataset was of high quality and suitable for further analysis and modeling.
